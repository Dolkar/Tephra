tp::ImageView class

Represents the view of a subresource range of tp::Image.

Contents

Optionally, it can be made to interpret the data in a different format than that of the viewed tp::Image or with a different tp::ComponentMapping.

Function documentation

ImageView tp::ImageView::createView(ImageViewSetup subviewSetup)

Creates another view of the viewed image relative to this view.

Parameters
subviewSetup The setup structure describing the view.

Remark

The subresource range of the new view must be fully contained inside this view.

This is effectively a shorthand for tp::Image::createView. This means that it isn't thread safe with respect to the underlying image. For job-local image views, this extends to the parent tp::JobResourcePool.

ImageSubresourceRange tp::ImageView::getWholeRange() const

Returns the image subresource range encompassing the entire range of the image view.

Remark

The returned range is relative to the view, therefore tp::ImageSubresourceRange::baseMipLevel and tp::ImageSubresourceRange::baseArrayLevel are always 0. To resolve the actual offsets to the underlying image, see tp::ImageView::resolveUnderlyingImage.

VkImageViewHandle tp::ImageView::vkGetImageViewHandle() const

Returns the associated Vulkan VkImageView handle if it exists, VK_NULL_HANDLE otherwise.

Remark

If the viewed image is a job-local image, the VkImageView handle will exist only after the tp::Job has been enqueued.

VkImageHandle tp::ImageView::vkResolveImageHandle(uint32_t* baseMipLevel, uint32_t* baseArrayLevel) const

Resolves and returns the underlying VkImage handle of this view or VK_NULL_HANDLE if it doesn't exist.

Parameters
baseMipLevel An output parameter to which the base mip level of the view will be set if the underlying image exists.
baseArrayLevel An output parameter to which the base array level of the view will be set if the underlying image exists.

Remark

If the viewed image is a job-local image, the underlying VkImage handle will exist only after the tp::Job has been enqueued.
